# 디렉터리의 구조

    - 디렉터리의 개념

        관련 있는 파일을 하나로 모아놓은 곳

        하나의 디렉터리에는 여러 개의 파일과 자식 디렉터리 존재할 수 있음

    - 디렉터리 파일

        디렉터리도 파일이고 파일 정보가 담겨 있음

        헤더를 가지고 있으며 디렉터리 정보가 시작되는 위치, 이름 등의 정보를 가지고 있음

        디렉터리 파일에는 자기 자신의 디렉터리, 하위/상위 디렉터리를 가리키는 파일을 가지고 있으며 파일에는 블럭 번호 있음

    - 경로

        파일이 전체 디렉터리 중 어디에 있는지 나타내는 정보

        - 절대 경로

            루트 디렉터리를 기준으로 파일의 위치를 나타내는 방식

        - 상대 경로

            현재 디렉터리를 기준으로 파일의 위치를 나타내는 방식

    - 디렉터리 구조

        초기 디렉터리는 1단계 구조

        파일이 많아지면서 트리 구조인 다단계 트리(순환이 없는) 구조

        오늘 날에는 바로가기 등이 있는 순환이 있는 디렉터리 구조

    - 마운트

        여러 개의 파티션을 붙이는 명령어

        각 파티션은 별도의 파일 테이블이 존재함

        파티션의 루트 디렉터리를 다른 파티션의 /mnt(마운트 연결 지점) 디렉터리에 붙이는 통합하는 방식
